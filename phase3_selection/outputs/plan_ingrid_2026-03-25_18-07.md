## Script inventory

### 01_process_all.py -- Full event selection + Lund plane construction
- Process ALL 6 data files and ALL 40 MC files (reco + gen + genBefore)
- Apply event selection: passesAll, Thrust > 0.7, N_ch >= 5, N_hemi >= 2
- Apply track cuts: pwflag==0, p > 200 MeV, |d0| < 2 cm, |z0| < 10 cm
- Split hemispheres using energy-flow thrust axis (TTheta, TPhi)
- Cluster each hemisphere with C/A (ee_genkt_algorithm, p=0, R=large)
- Follow primary declustering chain (p_T ordering w.r.t. beam)
- Record (ln 1/Delta_theta, ln k_T) for each primary splitting
- For MC: process reco, gen (tgen), and genBefore (tgenBefore) levels
- Prototype on 2-3 MC files first (~50k events), then full production
- Use ProcessPoolExecutor for parallel file processing if >2 min/file
- Output: NPZ files with Lund arrays, hemisphere counts, cutflow counts
- Separate cutflow reporting for data and MC

### 02_correction_infrastructure.py -- Bin-by-bin + IBU response matrix
- Bin-by-bin correction factors: C(i,j) = N_genBefore(i,j) / N_reco(i,j)
- IBU response matrix: 100x100 migration probability matrix
  - For each MC event, histogram gen-level splittings into gen bins,
    reco-level into reco bins (bin-level matching, NOT object-level)
  - Normalize rows to sum <= 1 (migration probability)
- Diagonal fraction per bin
- Quick prototype on small MC subset (~10k events) to check diagonal > 50%
- Output: NPZ with correction factors, response matrix, diagonal fractions

### 03_closure_test.py -- Mandatory, blocking
- Apply bin-by-bin correction to MC reco -> should recover MC genBefore truth
- Compute chi2/ndf with Poisson uncertainties (diagonal first)
- Compute p-value; must be > 0.05
- If fails: attempt 3 remediation approaches:
  1. Adjust binning (merge low-population bins)
  2. Apply smoothing to correction factors
  3. Use IBU instead
- Output: closure test results JSON

### 04_approach_comparison.py -- Approach A vs Approach C
- Approach A: thrust hemispheres (primary, from 01_process_all.py)
- Approach C: exclusive kT 2-jets (Durham algorithm)
  - Cluster full event with kT, require 2 jets
  - Use each jet as input to C/A declustering + Lund plane
- Compare Lund plane densities quantitatively (chi2, bin-by-bin ratio)
- Output: NPZ with Approach C Lund plane, comparison metrics

### 05_data_mc_lund.py -- Data/MC comparisons (correction gate)
- Reco-level Lund plane: data vs MC (2D comparison)
- 1D projections: ln k_T and ln 1/Delta_theta with ratio panels
- Hemisphere-level: multiplicity, sum-p_T, thrust
- All variables entering the observable -- hard gate before corrections
- Output: NPZ with comparison data

### 06_plot_all.py -- All figures (separate from analysis)
Figures to produce:
1. Cutflow bar chart (data + MC)
2. Reco-level 2D Lund plane (data)
3. Reco-level 2D Lund plane (MC reco)
4. Data/MC 2D ratio
5. Data/MC 1D projections (ln k_T) with ratio panel
6. Data/MC 1D projections (ln 1/Delta_theta) with ratio panel
7. Hemisphere multiplicity data/MC with ratio panel
8. Hemisphere sum-p_T data/MC with ratio panel
9. Correction factor map C(i,j) (2D)
10. Response matrix (2D migration probability)
11. Diagonal fraction map
12. Closure test: corrected MC reco vs MC gen truth (1D projections + 2D)
13. Closure test pulls
14. Approach A vs C comparison (2D ratio + 1D projections)

All figures: figsize=(10,10), CMS style, ALEPH label, PDF+PNG, no titles,
no absolute fontsize, mpl_magic for legends, make_square_add_cbar for 2D.
