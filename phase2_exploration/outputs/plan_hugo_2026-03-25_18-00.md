## Execution order

The deliverables are grouped into scripts. Each script is self-contained and
writes outputs (figures, log entries, data) to the appropriate directories.
Scripts are numbered by dependency order; independent scripts can run in parallel.

### Script 01: sample_inventory.py
**Deliverables:** 1 (sample inventory), 7 (aftercut pre-selection), 8 (thrust axis source), 9 (momentum resolution)
- Open one data file and one MC file with uproot, enumerate all trees and branches
- For each tree: name, number of entries, branch names and types
- Print unique values of all flag/weight branches (pwflag, passesAll, etc.)
- Count events per data file and per MC file
- Compare data Thrust vs ThrustCharged to determine thrust axis source
- Check d0 distribution width to diagnose TPC-only vs TPC+ITC+VDET
- Check tgenBefore scope: count events, compare to tgen
- Log all findings to experiment_log.md

### Script 02: data_quality.py
**Deliverables:** 2 (data quality)
- Load ~5000 events from one data file
- Check all particle-level branches for: NaN/inf, unphysical values,
  empty arrays, outliers (p < 0, theta outside [0, pi], etc.)
- Check event-level quantities: Thrust in [0.5, 1], N_ch > 0, etc.
- Check for year-to-year consistency (compare distributions from 1992 vs 1995)
- Report all findings

### Script 03: data_mc_comparisons.py
**Deliverables:** 3 (data/MC comparisons)
- Load ~50k events from data, ~50k from MC (reco level)
- Apply basic track selection: pwflag==0, p > 200 MeV/c, |d0| < 2 cm,
  |z0| < 10 cm, charge != 0
- Apply event selection: passesAll flag
- Plot with ratio panels (data as errorbar, MC as filled histogram):
  - Track p (momentum magnitude)
  - Track theta
  - Track phi
  - Track pt (transverse momentum)
  - N_ch per event (after track cuts)
  - Thrust
  - Hemisphere multiplicity
  - Opening angle between leading tracks in each hemisphere
- All plots follow appendix-plotting.md rules

### Script 04: cutflow.py
**Deliverables:** 4 (baseline yields / cutflow)
- Process ALL data and MC events (counting only, no arrays needed)
- Cutflow table: total -> passesAll -> thrust > 0.7 -> N_ch_hemi >= 2
- Report data and MC event counts at each stage
- Compute efficiencies

### Script 05: lund_plane_reco.py
**Deliverables:** 5 (reco-level Lund plane)
- Load ~100k data events (after selection)
- Apply track + event selection
- For each event, split into thrust hemispheres
- Cluster each hemisphere with C/A using fastjet.ee_genkt_algorithm(9999.0, 0.0)
- Follow primary declustering chain (harder subjet by p_T w.r.t. beam)
- Record (ln 1/Delta_theta, ln k_T) for each splitting
- Plot 2D Lund plane density as colormesh with colorbar

### Script 06: binning_optimization.py
**Deliverables:** 6 (binning optimization)
- Use MC matched events (tgen and t trees)
- For each MC event, build Lund plane at gen and reco level
- Compute resolution: sigma(reco - gen) for ln(1/Delta_theta) and ln(k_T)
  as function of the coordinate value
- Compute migration fraction per bin (off-diagonal population)
- Compute bin population for proposed 10x10 binning
- Compare to finer/coarser binning alternatives
- Compare uniform vs non-uniform options

### Script 07: pt_vs_energy_ordering.py
**Deliverables:** 10 (p_T vs energy ordering comparison)
- Use MC gen-level (tgen tree)
- Build Lund plane with p_T ordering and with energy ordering
- Compare the two 2D densities (ratio plot)
- Quantify the difference

### Script 08: tectonic_test.py
**Deliverables:** 11 (PDF build test)
- Create minimal stub .tex
- Compile with tectonic
- Delete stub
